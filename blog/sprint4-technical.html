Problem Solving

A story when I got blocked on a simple problem

I was trying to solve this coding challenge that involves using a simple loop. I tried to run my code, but it stops due to an error. I am a bit confused as to why it isn’t running since I am confident that my code is correct. I've been writing loops all my life! I tried to run it again and again – the same error keeps appearing.

Then I thought to myself, hey, why don’t you try to read what the error is saying. I read the error log and that’s when I realized that it’s just a syntax error, meaning that I used comma instead of semi-colon in my code. Correcting this issue made by code run perfectly.

I felt silly that the reason I was having problems is because of a careless mistake. I now have an appreciation for error logs and I know that they can help me solve my problem.

A story where I solved a problem in an elegant way

I was doing this other coding challenge wherein I was given a list of numbers and I need to remove the odd-numbered ones. I originally plan to solve this by writing a code that goes through each number on the list, check them one by one if they are odd, and remove them if they are. But then I looked at instructions again and it says that I can solve it either manually or using any of Javascript’s built-in function. So I searched the list of built-in functions and I was able to find one – filter – that exactly does what I am trying to do.

It made me happy that I was able to find a more elegant solution as it made my codes much cleaner. It also reminded me of the concept of Einstellung which means that prior knowledge can be a barrier to creative problem solving. I am so used in using Excel and this makes my initial response in figuring out a solution to a coding problem is how I’m going to solve it in Excel.

Problem solving techniques

The following problem solving techniques are the ones that I routinely used during this sprint’s coding challenge. I can say that I am confident in using them:

  • Pseudocode
  • Trying something
  • Reading error messages
  • Console.logging
  • Googling

However, as the techniques above were mostly sufficient to get me through the challenges, I was not able to practice the techniques below, so I am not as confident in using them:

  • Rubber ducky method
  • Asking your peers for help
  • Asking coaches for help
  • Improving your process with reflection
